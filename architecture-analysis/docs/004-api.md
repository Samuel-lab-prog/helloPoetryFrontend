# API Layer

This document describes how API communication is organized and how to evolve it safely.

---

## Location

All API clients live under:

```
core/api/
```

Each domain area (auth, poems, users, etc.) has its own module.

---

## Responsibilities

API modules are responsible for:

- defining request/response shapes
- performing HTTP calls
- returning typed results
- centralizing API concerns (headers, base URL, error handling)

API modules must not:

- contain UI logic
- mutate feature-specific state directly

---

## Generated Clients

The project uses `orval` to generate typed clients from the backend OpenAPI spec. Generated code
must be treated as **read-only** and extended through wrapper modules if additional behavior is
needed.

Script:

```
bun run generate:api
```

---

## Error Handling

Errors from API calls should be normalized in `core/api` and surfaced as typed results so features
can handle them consistently.

---

## Dependency Direction

- Features depend on `core/api`.
- `core/api` never depends on features.
